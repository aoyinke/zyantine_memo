1. 问题分析：

   * 当 memo0 记忆系统初始化失败时，`self.memory` 被设置为 `None`

   * `get_recent_conversations` 方法直接调用 `self.memory.search()` 和 `self.memory.get_all()`，没有检查 `self.memory` 是否为 `None`

   * 这导致了 `AttributeError: 'NoneType' object has no attribute 'search'` 和 `AttributeError: 'NoneType' object has no attribute 'get_all'` 错误

2. 修复方案：

   * 在 `get_recent_conversations` 方法中添加对 `self.memory` 是否为 `None` 的检查

   * 如果 `self.memory` 为 `None`，使用简化的内存存储逻辑（就像 `search_memories` 方法一样）

   * 否则，继续使用原有的 memo0 API 调用

3. 具体实现：

   * 在 `get_recent_conversations` 方法开始处添加 `if not self.memory:` 条件判断

   * 在条件块中实现基于 `self.memory_store` 的最近对话获取逻辑

   * 保持原有的错误处理和回退机制

4. 预期效果：

   * 当 memo0 记忆系统初始化失败时，系统会自动切换到简化的内存存储模式

   * 不再出现 `AttributeError` 错误

   * 系统能够正常获取最近对话
